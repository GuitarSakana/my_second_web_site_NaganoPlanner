# 직접 개발하기 어려운 기능들은 구글링을 통해 깃허브에서 다른 개발자들의 오픈소스코드나 라이브러리를 가져와서 사용하였다.
   그렇게 가져온 소스코드들을 전부 하나하나 뜯어서 분석한 후, 내 입맛에 맞게 코드를 수정해가며 디자인을 바꾸고 기능을 빼거나 더해서 사용했다.
   ex) mp3player, calender 등등

# 분석한 소스코드에 주석으로 코드들의 흐름이나 기능들에 대해 설명을 달아두었다. (내 소스 코드는 구분이 쉽게 주석처리 하며 개발)




#느낀점

       -  구글링과 chat-gpt를 적극 활용하면서 내게 필요한 정보들을 효과적으로 잘 찾을 수 있게 되었다. 
       -  그렇게 찾아낸 정보들을 내 프로젝트(코드)에 적절하게 적용할 수 있게 되었다.
       -  open API의 엄청난 힘과 그 힘을 어떻게 사용하면 좋을지 사용방법을 조금이나마 알게 되었다.
       -  맨땅에 헤딩하듯이 처음부터 전부 하나하나 코드를 짜려니 도움은 많이 됐지만 효율이 너무 떨어짐을 느꼈다. 프레임워크 공부의 필요성을 절실히 느꼈다.
